# EduPractica: explicación general (versión para presentación)

## 1. ¿Qué es EduPractica?

EduPractica es una plataforma educativa digital que ayuda a organizar, compartir y dar seguimiento al aprendizaje por etapas.

En palabras simples:
- Los **estudiantes** aprenden paso a paso.
- Los **profesores** crean y comparten contenido.
- Los **administradores** supervisan calidad, usuarios y resultados.

El objetivo principal es que el aprendizaje sea **ordenado, medible y colaborativo**.

---

## 2. ¿Qué problema resuelve?

Antes de un sistema como este, normalmente hay:
- Materiales dispersos.
- Poca visibilidad del avance real de los estudiantes.
- Dificultad para mantener calidad del contenido.
- Riesgo de perder recursos cuando un docente deja de participar.

EduPractica resuelve esto centralizando contenido, controlando su calidad y mostrando resultados en tiempo real.

---

## 3. ¿Cómo funciona de forma general?

El sistema está dividido en dos partes:

### Frontend (lo que ve el usuario)
Es la aplicación web donde cada perfil trabaja:
- Estudiante: ruta de aprendizaje y progreso.
- Profesor: gestión y transferencia de materiales.
- Administrador: control de usuarios, contenidos y métricas.

### Backend (lo que procesa internamente)
Es el núcleo que:
- Valida accesos y roles.
- Guarda y organiza la información.
- Aplica reglas del negocio.
- Entrega reportes y datos para toma de decisiones.

---

## 4. Funcionalidades clave (en lenguaje de negocio)

- **Inicio de sesión seguro** (incluye acceso social).
- **Gestión de cursos/categorías y etapas**.
- **Progreso del estudiante por niveles** (desbloqueo secuencial).
- **Revisión y aprobación de contenido** antes de publicarlo.
- **Pistas y retroalimentación** durante actividades.
- **Panel de resultados y reportes** para seguimiento académico.
- **Transferencia de contenido entre profesores** para asegurar continuidad.

---

## 5. Beneficios para la institución

- Mayor control y orden académico.
- Mejor experiencia de aprendizaje para estudiantes.
- Mejor coordinación entre profesores y administración.
- Datos concretos para decisiones pedagógicas.
- Menor dependencia de una sola persona para conservar contenido.

---

## 6. Stack tecnológico (explicado sin profundidad técnica)

- Se usa una **aplicación web moderna** para la interfaz (rápida, adaptable y multilenguaje).
- Se usa un **servicio backend robusto** para seguridad, reglas y datos.
- Se cuenta con herramientas para **analítica y generación de reportes** (Excel/PDF).

---

## 7. Relación con prácticas de ingeniería solicitadas (versión ejecutiva)

### Scrum
En el proyecto se utilizó Scrum para trabajar por entregas cortas y funcionales, organizando el avance por módulos y mostrando resultados continuos en cada ciclo.

### Procesos, tareas y modelo MM
Se definieron procesos y tareas por rol, y se aplicó un enfoque MM para medir resultados y sostener la mejora continua del producto.

### ADOO
Se aplicó ADOO para estructurar la solución alrededor de objetos de negocio (usuarios, etapas, feedback y transferencias), logrando mayor orden, claridad y escalabilidad.

### Análisis de modelos de descripción
Se realizó análisis de modelos de descripción para asegurar coherencia entre lo definido a nivel funcional, los datos gestionados y los procesos implementados en la interfaz.

### Análisis situacional y documental
Se desarrolló un análisis situacional y documental para alinear la solución con necesidades reales del entorno educativo y respaldar su implementación con documentación útil para operación y mantenimiento.

### Definición de requerimientos
Se definieron requerimientos funcionales y no funcionales desde etapas tempranas, estableciendo reglas de operación y criterios de calidad que guiaron el desarrollo.

---

## 8. Mensaje final para presentación

EduPractica no solo digitaliza contenido; también crea un ecosistema de aprendizaje con control de calidad, trazabilidad y mejora continua. Es una solución pensada para crecer con la institución y respaldar decisiones académicas con evidencia.
